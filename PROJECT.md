# PROJECT

NSJ Lock 是一个 Windows 桌面音量保护工具。它的核心目标是保护系统默认输出设备的主音量，既支持把音量稳定锁定在用户设定的百分比，也支持 Sound Lock 风格的动态限制，避免系统、应用、会议或误操作带来突然过响的声音。

## 产品目标

- 提供一个轻量、可长期后台运行的 Sound Lock 风格工具。
- 在保护开启时，提供固定锁定和动态限制两种保护模式。
- 固定锁定模式将系统主音量强制维持在设定值。
- 动态限制模式允许用户继续调系统音量，只在输出峰值过高时临时压低。
- 保持本地优先：不需要账号、不上传数据、不做云同步、不接入分析统计。
- 以单文件 Windows x64 exe 发布，方便下载、替换和回退。

## 目标用户

- 希望固定电脑系统音量的人。
- 希望开会时能调高系统音量听清小声人声，同时避免突然爆音的人。
- 经常遇到应用、会议、游戏或系统事件改动主音量的人。
- 需要一个简单、可见、可退出的本地桌面工具，而不是复杂音频处理器的人。

## 当前范围

NSJ Lock 当前只负责系统默认输出设备的主音量保护。

已包含：

- 系统主音量锁定到指定百分比。
- 音量高于或低于目标值时恢复到目标值。
- 动态限制模式下读取输出峰值，并在峰值过高时临时压低系统主音量。
- 动态限制模式下声音恢复安全后，逐步恢复到用户目标音量。
- 显示当前默认输出设备和当前系统主音量。
- 显示当前输出峰值。
- 本地 JSON 设置保存。
- 托盘后台运行与托盘退出。
- 深色、浅色、跟随系统主题。
- 中文和英文界面切换。

明确不包含：

- 单应用音量控制。
- 虚拟声卡、底层音频流 DSP、多段压缩或复杂音频处理。
- 参会者级别的单独音量控制。
- 会议软件 API 集成。
- 账号、云同步、远程配置或分析统计。
- 自动更新器。

## 架构边界

解决方案分为三个主要项目：

- `src\NSJLock.App`：WPF 应用、主窗口、托盘、命令、界面文本和 ViewModel。
- `src\NSJLock.Audio`：音频设备访问、默认输出设备读取、音量保护逻辑。
- `src\NSJLock.Config`：本地设置模型、JSON 读写和配置默认值。

测试集中在：

- `tests\NSJLock.Tests`：配置、音频保护逻辑、音频端点包装和 ViewModel 行为测试。

架构原则：

- 音频控制属于高风险路径，修改前先收窄目标和验证方式。
- UI 调整不能顺手改变音频主链路。
- 配置模型改动必须考虑旧设置文件兼容。
- 发布脚本和公开导出脚本属于交付边界，修改时单独验证。

## 数据与隐私

NSJ Lock 的默认数据边界是用户本机。

- 设置文件保存在 `%AppData%\NSJ Lock\settings.json`。
- 仓库不应引入后端、账户系统、遥测、日志上传或云同步，除非用户明确改变产品方向。
- 发布产物、SDK、NuGet 缓存和临时目录不得进入公开仓库。

## 发布边界

版本号以 `src\NSJLock.App\NSJLock.App.csproj` 中的版本字段为准。

发布统一通过：

```powershell
powershell.exe -ExecutionPolicy Bypass -NoProfile -File .\scripts\publish-single-file.ps1
```

发布产物固定在：

```text
artifacts\publish\NSJLock-single-win-x64
```

`NSJLock.exe` 代表最新版，`NSJLock-v<版本号>-win-x64.exe` 代表可归档和上传 GitHub Releases 的版本文件。
